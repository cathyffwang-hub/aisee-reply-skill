# aisee-reply-skill

## Description

AiSee 反馈自动回复工具。适用于腾讯文档企业版 AiSee 反馈平台的定期回复场景。

触发关键词：aisee 回复、反馈回复、处理 aisee 问题、打开回复工具、运行回复流程

## Capabilities

- 自动抓取 AiSee **目标日期当天**（默认昨日）未回复的用户反馈（严格按日期过滤）
- 关键词命中模板 B/C/A 自动回复
- **其余问题由 AI 根据知识库生成高质量回复**（零 API 成本，利用加载 Skill 的 AI 自身能力）
- 生成可编辑回复确认网页（答案可改，点击确认即实时驱动浏览器回复，状态 localStorage 落盘）
- 自动检查并确保静态服务从 skill 目录正确启动
- 企业微信通知（@指定账号 + 暖心话 + 工具链接），**同一天只推送一次（去重）**
- 知识库包含 15 个腾讯文档企业版官方文档，90 天自动刷新

## Usage

触发此 Skill 时，**严格按以下两阶段流程执行**：

---

### 第一阶段：抓取与预处理（自动）

执行 `node run.js` 完成以下步骤：

1. **确保静态服务**（端口 3399，根目录为 skill 目录）
2. **检查知识库**（`memory/knowledge.md`，过期自动刷新）
3. **iOA 登录**（已有 session 直接跳过，未登录自动发起验证轮询等待）
4. **抓取反馈并过滤**（只取 targetDate 当天 + 未回复）
5. **关键词预处理**：
   - 含「企微/企业微信/企微文档」→ 模板C（自动填入）
   - 含「会员/退费/发票/开票/充值/付费/订单/退款/vip」→ 模板B（自动填入）
   - 中文字符 < 4（外文/无意义）→ 模板A（自动填入）
   - **其余问题**：标记 `needsAI: true`，`answer` 留空
6. **输出** `output/pending_ai.json`

---

### 第二阶段：AI 生成回复（核心！）

**这一步由加载此 Skill 的 AI 助手执行，不需要任何外部 API。**

#### 步骤

1. **读取** `output/pending_ai.json`，找出所有 `needsAI: true` 的条目
2. **读取** `memory/knowledge.md` 作为知识库上下文
3. **逐条生成回复**，严格遵循以下规则：

#### AI 回复生成规则（必须遵守）

**角色定位：** 你是腾讯文档企业版的客服专员，回复用户在 AiSee 反馈平台提交的问题。

**回复原则：**
1. **优先从知识库找答案**：仔细阅读 `memory/knowledge.md` 中的全部内容，找到与用户问题最相关的功能说明或操作指引
2. **给出具体的操作步骤**：不要泛泛而谈，要告诉用户具体怎么操作（点哪里、选什么、步骤123）
3. **语气专业友好**：以「您好！」开头，语气亲切但专业
4. **知识库无答案时**：如果问题确实超出知识库范围（如 bug、服务异常），承认问题已记录，告知会安排跟进，建议联系企微小助手
5. **不要编造功能**：只能基于知识库中明确记载的功能回答，不确定的不要瞎说
6. **回复长度**：100-300字，不要太长也不要太短

**回复格式参考：**
```
您好！关于您反馈的[问题概述]，[具体解答/操作步骤]。如有其他问题，欢迎继续反馈。
```

**示例：**
- 问题：「如何快速查找协作者修改的内容（表格比较大的情况）」
- 回复：「您好！关于查找协作者修改内容的问题，您可以使用以下方式：1）在文档右上角点击「文档操作」→「版本及编辑记录」，可查看修订记录，包含版本、修订时间、修订人等信息；2）在线文档还支持开启「修订模式」（顶部工具栏→审阅→修订模式），开启后所有修改会以高亮形式呈现，新增内容显示为蓝色下划线，删除内容显示为红色删除线，不同协作者用不同颜色区分。」

4. **将生成的回复写入** `output/pending_ai.json` 中对应条目的 `answer` 字段
5. 同时将 `tag` 改为 `'ai'`，`tagLabel` 改为 `'AI智能回复'`，`needsAI` 改为 `false`

---

### 第三阶段：生成 HTML 并推送（自动）

执行 `node run.js --build [targetDate]` 完成：

1. 读取已填充的 `output/pending_ai.json`
2. 生成回复工具 HTML（`output/reply_tool.html`）
3. 保存数据快照
4. 企微通知推送（去重）

---

### 完整调用示例

```bash
# 第一阶段：抓取问题（自动）
cd ~/.workbuddy/skills/aisee-reply-skill && node run.js

# 第二阶段：AI 读取 pending_ai.json + knowledge.md → 填充回复（由 AI 执行）

# 第三阶段：生成 HTML 并推送（自动）
cd ~/.workbuddy/skills/aisee-reply-skill && node run.js --build 2026-04-13
```

## File Structure

```
aisee-reply-skill/
├── SKILL.md              # 本文件（Skill 执行规范，AI 必读）
├── README.md             # 安装和配置说明
├── run.js                # 主执行脚本（两阶段模式）
├── reply_server.js       # 浏览器驱动回复服务（端口3400）
├── memory/
│   ├── knowledge.md      # 知识库（15个文档源，90天自动刷新）
│   ├── notify_lock.json  # 企微通知去重锁
│   └── snapshot_YYYY-MM-DD.json  # 数据快照
└── output/
    ├── pending_ai.json   # 第一阶段输出：含待 AI 填充的问题列表
    └── reply_tool.html   # 最终回复工具网页
```

## Configuration

在 `run.js` 的 `CONFIG` 对象中修改：

| 字段 | 说明 |
|------|------|
| `AISEE_LIST` | AiSee 列表页 URL |
| `AISEE_DETAIL` | AiSee 详情页 URL |
| `WECOM_WEBHOOK` | 企业微信群机器人 Webhook 地址 |
| `WECOM_MENTION_USERID` | 企微 @人的 userid（当前：`cathyfwang`） |
| `KNOWLEDGE_DOCS` | 知识库文档源列表（数组，含 id/url/title） |
| `KNOWLEDGE_REFRESH_DAYS` | 知识库刷新周期（默认90天） |
| `REPLY_PORT` | 回复服务端口（默认3400） |
| `STATIC_PORT` | 静态服务端口（默认3399） |
