💹 如何使用仪表盘1. 功能简介        仪表盘是一个数据可视化容器，让您能够轻松地将各种图表添加到智能表格中，从而实现对数据的多维度分析和呈现。通过仪表盘，您可以快速创建精美的业务图表，实时跟踪关键数据指标，从不同角度洞察数据变化，提升企业决策效率和质量。2. 操作流程step1：创建仪表盘 创建仪表盘：打开一份智能表格，点击左下角加号键，选择「新建仪表盘」（注：5月14日后新建智能表格，下方会自动出现「仪表盘」工作表），点击「添加图表」，选择「图表类型」，配置样式，选择一个数据表作为数据源，定义图表数据对应的具体字段，即可开始创作之旅。查看仪表盘：点击左上角「筛选」，添加筛选条件，即可得到定制化看板发送仪表盘：点击左上角「设置自动化发送」，可设定在添加新纪录/记录内容修改/定时重复/到达记录中时间时向指定文档用户推送仪表盘，仪表盘将以图片形式展示step2：选择不同仪表盘图形 👀 在仪表盘中，图表的横轴通常代表需要观察的维度，例如时间、地区、产品等，而纵轴则代表需要观察的指标，例如销售额、用户数量、转化率等。📈 根据不同的业务需求和数据特性，可以选择不同类型的图表进行展示，例如柱状图、条形图、折线图、散点图、饼状图等。每种图表都有其特定的应用场景和表达方式。👍 使用图表，您可以直观地了解数据的概览情况，从而做出更加精确和有效的决策。同时，图表也是会议汇报中的重要工具，可以帮助您更清晰、更生动地向他人展示数据分析的结果。|  |  |  |  |  |

2.0 sheet 为什么用不了跨文件引用函数呢？     由于跨文件引用函数技术架构设计问题，目前是只能在 3.0 sheet 上进行使用的。存量用户 2.0 sheet暂不能体验。跨表引用的数据会自动同步源数据吗？     当你跨表引用的源表(被引用的表格)数据发生变更时候，那么IMPORTRANGE公式所在表格将会自动同步源数据的变更，保证你能随时查看最新数据信息。我可以直接修改引用后的数据吗？     不可以，如果你在IMPORTRANGE公式引用数据成功后，想直接修改这部分跨表引用的数据是不允许的，会报错 #SPILL!。跨文件引用能引用智能表格吗？     跨文件引用功能不能引用智能表格数据。如何区分3.0 还是 2.0 表格？      可以让用户在表格输入"="，如果可以看到 IMPORTRANGE函数的提示，说明是3.0 表格。哪些表格文档可以被我引用呢？这里就涉及权限安全相关问题，首先你需要具备被引用文档的大表权限的“可复制权限”。1.如果你引用的是自己创建的表格，那么无论表格本身是什么权限（私密或者指定协作人可查看），你都是有权限引用的。     2.如果你引用的是别人创建的表格。   2.1 如果该表格是私密文档（仅我可查看），不管开不开启高级权限中的复制权限都是不能引用的。   2.2 如果该表格是指定协作人权限，先看是否在高级权限中开启了禁止复制（禁止查看复制、禁止编辑复制），如果开启了，那么是没有引用权限的，如果没有开启，那么看你是否在协作人列表，如果没有在协作人列表中，也是没有引用权限的。   2.3 如果该表格是所有人可查看权限，先看是否在高级权限中开启了禁止复制（禁止查看复制、禁止编辑复制），如果开启了，那么是没有引用权限的，如果没有开启，那么是有引用权限的。   2.4 如果该表格是所有人可编辑权限，先看是否在高级权限中开启了禁止复制（禁止查看复制、禁止编辑复制），如果开启了，那么是没有引用权限的，如果没有开启，那么是有引用权限的。当你具备被引用文档的大表权限后，还需要看你是否具备具体引用格子的区域权限。     1.如果你没有设置区域权限，那么肯定是可以引用的。     2.如果设置了区域权限。【这里待确认下，目前有点出入】2.1 仅我可编辑。2.2 指定人可查看、可编辑。如果报错提示没有引用表格的权限，怎么办呢？一般有以下两种情况会提示用户没有引用表格的权限。      1.用户写IMPPORTRANGE时候没有文档的引用权限；      2.用户开始时候有引用权限，后续权限被回收了。      用户只需要按照报错提示，有按钮点击，跳转到权限申请页面，进行权限申请即可，权限审批通过，即可自动正常引用。🗂如何使用智能分列1. 功能简介 先在文档中把输入光标定位到需要插入链接的位置，在顶部工具栏中选择【插入—链接】或者输入快捷健【Ctrl+k】，然后在提示框中输入对应内容即可。方式二：在已有文本插入链接        选中文档中需要添加超链接的内容后，界面会自动弹出工具栏，点击 【链接】图标，然后在提示框中输入对应内容即可。3. 应用场景        某个议题头脑风暴会议中，可以在提前准备的会议提纲中插入网页内容作为讨论的依据，会议进行中直接点击即可跳转链接，无需再次打开浏览器搜索或打开网页，为整体会议节奏提升效率。

五、智能表格
「智能表格」本质上是一款可以多人在线协同的关系型数据库，只是其交互设计采用类似在线表格的方式。因此，智能表格同时兼备了业务系统对结构化数据的展示、分析、流转、精细权限控制能力和普通用户对在线表格的低认知门槛。在数据安全保护方面，智能表支持区分子表、分行列、分字段进一步设置内容权限，保障企业数字资产安全。智能表目前支持最高10w行、1500w单元格编辑。

1.1 新建空白智能表
支持在在线表格内新建空白智能表，点击左下角「+」可以新建。

1.2 生成智能表
支持将现有在线表格直接生成智能表，右键在线表格底部，即可生成。

1.3 用所选数据新建智能表
支持在在线表格中选中数据，右键点击「用所选数据新建智能表」即可新建智能表。

1.4 首页新建智能表
支持在文档首页直接新建智能表格。

2.1 「列」类型
智能表格支持的列类型目前有22种，每一种列类型都对列的内容有严格的限制，同时也有适用于该类型的属性可供设置。  
支持将成员姓名或用户名称复制到「人员」列时，自动匹配该人员。（若该姓名有重名，会随机匹配人员）

点击列头处的下三角，或在列头处右键单击，在弹出的菜单选项中点击「设置列类型」，可以打开列设置面板，设置该列的列类型，比如文本、图片、单选等类型。

新增「公式模板库」，提供上百种公式模板（如计算排名、重复值标记、逐日累计等），无需写公式即可实现不同场景下的数据计算。

在线PDF支持AI 阅读。可对PDF内容智能进行总结，同时可将总结内容一键转为新的文档；支持根据PDF内容生成可编辑的思维导图；支持基于PDF内容进行问答式分析，可一键润色提问指令，并支持将回答内容转为新的文档。
  


十二、共享空间
「共享空间」是主要用于团队文档共享管理，共享空间可以关联部门，并可以对部门成员进行多层级权限管理，通过共享空间可以快速搭建团队共享知识库，也能保证团队的文档安全。

1、创建共享空间
可手动为项目/团队创建共享空间，共享文档资料，也支持在管理后台配置同步组织架构的团队信息，自动关联和创建部门共享空间。

2、添加共享空间成员
支持以部门和成员维度添加企业成员为空间成员，成员添加好之后可以对不同的空间成员设置不同的权限，比如管理员、可编辑、仅预览、自定义权限、禁止访问等。

3、添加内容
支持在共享空间中添加子文件夹，并在不同空间中「新建」在线文档或「导入」本地文件，实时维护和更新共享空间知识。

4、权限管理与安全设置
权限管理可以添加空间成员，设置成员权限。安全设置可以针对共享空间的整体进行安全管控，比如保密模式、禁止文档分享到企业外等。

5、更多设置
支持在更多设置中，根据用户需求设置重命名、转让所有权、退出空间、解散空间等操作。


空间（V1.11新增）
腾讯文档企业版「空间」是专为企业打造的一站式团队知识管理与协同中心。它基于腾讯文档强大的多人实时协作能力，深度融合企业组织架构，提供集文档存储、知识沉淀、精细化权限管控与数据安全保障于一体的企业级内容协作平台。
「空间」的典型特征在于支持树状目录结构，允许创建“父文档”与“子文档”的层级关系，就像为一本书建立清晰的章、节、目，旨在逻辑性地组织知识，形成一个稳定、可全局检索且能持续完善的知识管理体系。
1、创建空间
支持在文档首页创建空间，设置空间名称、可见范围，以及添加空间协作人。

2、空间主页
支持在「空间」主页进行查看空间信息、复制空间链接进行分享、设置空间主页、管理空间权限，以及一键新建页面、添加已有文档等操作。


3、空间目录操作
支持在「空间」目录新建页面、新建文件夹等操作，并且对某个具体子页面或文件夹进行分享、权限管理、重命名、移出、删除等操作。（将文档「移出」空间后，文档将回到文档创建者的「云文档」中，并在文档创建者、其他空间成员的「消息通知列表-其他通知」中进行提醒）
   

十四、权限管理和安全管控
1、添加成员
可以在文档中以部门和成员维度添加企业成员进行文档协同，成员添加好之后可以对不同的文档成员设置不同的权限，比如管理员、可编辑、仅预览、自定义权限、禁止访问等。

2、分享文档
支持分享文档链接或生成二维码给企业成员，在分享时可设置企业成员文档权限。


3、浏览记录
可以查看浏览记录，浏览记录会记录浏览时间、浏览人、以及整体的浏览量。

4、修订记录
可以查看文档的历史修订记录，修订记录会记录版本、版本时间、修订人等信息，并且支持还原到某一个历史版本。

5、权限管理
文档支持精细化权限管理，具体可从加入规则、文档成员、安全设置三方面进行设置。

5.1 加入规则
支持设置分享范围和权限，可将文档分享给企业内指定的成员、企业内全部成员，并可对权限进行设置，确保资料分享安全可控。支持设置管理员审批或文档成员审批，并在申请权限的界面显示所有审批人的名字，审批人默认显示3个，超过3个显示"aa/bb/cc/..."，同类角色的成员默认按照姓名首字母进行排序。


支持对文档设置加入规则的有效期，当前可选择「1天/7天/永久有效」三种默认选项，同时可以自定义有效期，自定义周期范围没有时间限制。加入规则的时间过期后，默认变为需管理员审批。


5.2 文档成员
支持按部门、指定人员添加特定权限，转交所有权，实现精细化权限设置，满足特定情况下文档分享需求。

5.3 安全设置
可以在安全设置中开启文档水印，禁止预览成员发表评论，以及可以设置禁止所有成员或预览成员复制、打印、导出内容或创建副本，更加精细与严格的管控文档权限。

6、水印设置
文档支持设置水印，且可自定义水印信息与样式。

管理后台功能介绍

管理员可以「组织架构」中新建、修改或删除部门，添加、修改员工信息，对员工操作离职以及查看员工状态。

1.1 批量添加员工
管理员可以批量导入员工数据，选择「批量导入/导出」，可以在组织架构初始阶段，下载空白模板批量导入新成员，如需实现成员的批量更新，可以导出当前数据更新导入。

1.3 人员置顶
支持在「部门架构」中将人员置顶。

企业管理员可以设置组织架构可见范围，主要包含三种规则：隐藏部门/人员、限制查看外部部门、限制查看所有人，管理员可以根据部门或人员的使用场景选择适用的规则。

3、许可证管理
许可证管理可以查看和管理腾讯文档整体的许可使用情况，比如已启用许可证数、总购买许可证数、到期时间等，并且可以按照部门、人员调整腾讯文档授权范围。

6、文档管理
6.1 离职成员转交
成员离职后，管理员可以看到该员工的文档数量、是否已转交、转交文档数量，支持批量操作指定文档转交给其他员工，方便快捷的管理离职员工的文档数据。

6.2 容量管理
管理员可以查看腾讯文档整体容量使用情况，包含已使用、可用、使用率等数据，同时可以设置每个企业成员可使用的容量限制，并且支持自定义设置管理后台容量，让企业能更精准的控制企业员工的容量使用。

6.3 共享空间管理
腾讯文档支持共享空间管理，管理员可以修改空间的超级管理员，确保共享空间的管理权限得到正确的分配和控制。

6.4 企业回收站
企业回收站会存放180天内已删除的文档/文件夹、已解散的空间、已删除的离职成员文件，管理员可以在过期时间内恢复或彻底删除回收站中的内容，防止企业成员误操作后无法找回文档。

7、文档权限
7.1 文档默认权限
支持配置文档权限默认值。管理员可以设置企业内收到新建文档是否默认可编辑，开启默认可编辑时，可以设置指定成员仅浏览、成员审批、管理员审批对应新建文档的权限。

7.2 水印设置
管理员可以开启强制「文档水印」，同时可以设置水印信息与水印样式（访问者姓名、访问者手机号后四位、访问者账号、时间戳、工号），企业成员无法自行关闭，加强企业内部信息管理，防止截屏、拍照泄密。
 
7.3 分享链接管理
管理员可以在企业管理后台针对员工已分享文档设置取消分享，操作取消分享后，文档加入规则变更为「由管理员审批」。


7.4 共享空间基础权限
支持管理员设置禁止全员或指定成员创建共享空间，可以限制创建共享空间的最大空间人数、设置指定成员不受限制，同时可以开启自动创建部门空间，方便部门内部的文档协作和管理。

7.5 操作日志查询
管理员可以按照支持按时间、操作行为、名称、链接、用户id（组织架构用户账号信息里的用户id）进行筛选查询操作日志，同时也可以导出日志进行分析，便于对文档的操作记录进行追溯，保障企业资产安全。

7.6 敏感操作预警
对于异常大批量导出、下载文件等敏感操作行为，支持设置手机、邮箱发送告警，及时提醒管理员有可能存在的安全风险和数据泄露。

7.7 粘贴保护
企业管理员可以在后台设置整体文档环境「禁止成员将文档内容粘贴外部」，设置完成后，新规则需要3-5分钟生效。企业成员复制文档内容时出现弹窗提示，文档内容只能在企业所属的文档环境内进行粘贴，无法粘贴到外部环境，保障文档资产安全。

7.8 回调接口管理
建立流水日志平台，统一提供日志流水接入，企业可接入自己的大数据平台进行多维度数据统计和展示。（具体接口可以查看：HYPERLINK https://docs.qq.com/open/document/private/subscription/event/oplog.html normalLink \tdkey 6af6az \tdfe -10 \tdfn https%3A//docs.qq.com/open/document/private/subscription/event/oplog.html \tdfu https://docs.qq.com/open/document/private/subscription/event/oplog.html \tdlt inline https://docs.qq.com/open/document/private/subscription/event/oplog.html）

7.9 文档成员搜索权限
管理员可设置文档成员搜索权限。

可搜任何成员：不限制搜索权限，可搜索公司内任何成员



7.10 密级标签（V1.11新增）
支持对指定人员/部门添加密级标签，被打标签的成员创建的文档将自动附带保密标识（即：谁的文档会打上密级标签），基于已建立的密级标签，可进一步配置标签策略（即：密级标签用来限制谁的什么操作）
受标签影响的成员视角：1.  顶部自动显示密级标签名称（仅标签策略的受限人员可见）；2.  提示受策略限制的行为；3.  受策略限制的行为入口置灰，不可操作。

7.10.1 新建密级标签
新建密级标签，选择受密级标签管控人群范围（即：谁的文档会打上密级标签）。


7.10.2 新建标签策略
新建标签策略，设置命中密级标签的人员和限制规则（即：密级标签限制谁的什么操作）。



7.11 空间设置（V1.11新增）
支持在管理后台对「新建空间」的操作进行权限设置：
可限制「新建空间」的角色，选择仅企业管理员、企业内成员可创建（如企业内成员被限制了「新建空间」，文档首页「新建」菜单将隐藏创建入口，同时在「空间」主页右上角的「新建空间」入口将置灰不可点击）
可限制成员新建「企业内全员可见空间」的角色，选择禁止任何人新建、仅企业管理员新建、企业内成员（如企业内管理员/成员被限制了「创建全企业可见空间」，新建空间面板时「可见范围」 将置灰不可点击）

8、模版管理
8.1 模板配置
管理员可以对各品类文档模版做新增、导出、更新、下架等操作，发布成功的模板会自动同步到文档首页的模板库中。

8.1.1 新增模板
（1）先在文档首页导入本地文件，获取原始文档链接。

（2）管理员进入企业管理后台，在模板管理—模板配置页面，新增模板。

（3）填写新模板信息，填写好提交。

（4）模板信息提交成功后，将新增模板添加到专题，选择对应的专题，点击确定。

（5）关联专题后，新增模板先操作上架，再操作发布。

（6）模板发布成功后，进入专题管理界面，选择模板关联的专题，点击发布，即可在对应专题模板库中展现新增的模板。

8.2 专题管理
管理员可以创建模版专题，并增加专题相关的信息，便于对企业自有的模板进行分类管理，如按照职能、业务及常见使用场景来进行专题管理。

8.3 页面配置
管理员配置专题模板后，将「专题id」用于配置模板库中所展示的品类专题。
注：专题id在「专题管理」页面中获取。

密码策略

企业信息
管理员可以设置企业名称、企业LOGO、企业登录标识等企业信息，在文档登录页彰显企业品牌。

管理员角色
企业创建人默认拥有「超级管理员」的角色，超级管理员可以添加其他成员为超级管理员。同时，超级管理员可设置「腾讯文档管理员」，赋予后台管理权限。

支持创建不同角色的管理员，并区分管理员可管控范围。（管理后台的管理员分为超级管理员、管理员，超级管理员拥有管理后台的全部权限，管理员拥有部分权限。超级管理员可以添加管理员角色，如“组织架构管理员”、“办公套件管理员“等。同一个角色中的管理员拥有的权限相同。创建管理员角色并为其分配权限有助于管理员各司其职，降低企业管理成本。）


日志查询
14.1 用户日志
用户行为日志会记录普通用户在系统中的操作行为，包括登录、登出、修改密码等。管理员可以通过操作类型、操作时间查询用户行为日志，可以从日志中查询到操作者、操作详情、操作结果、IP等信息，同时可以导出日志进行详细分析。

14.2 管理员日志
管理员操作日志会记录管理员用户在系统中的操作行为，包括添加人员与部门，添加应用与认证源等。管理员可以通过操作类型、操作时间查询管理员操作日志，可以从日志中查询到操作者、操作详情、IP等信息，同时可以导出日志进行详细分析。

快速上手智能文档
步骤一：创建智能文档
方式一：新建空白智能文档
在文档首页，点击『新建』，页面会弹出不同类型的文档；选择『智能文档』品类，即可跳转智能文档页面，快速新建一篇空白智能文档。


方式二：从模板库创建智能文档
点击首页的『模板库』，或者选择新建智能文档（上述方式一）时，会进入到下图所示的『文档模板』页面，我们可以按需选择匹配度高的文档模板，将鼠标悬浮在对应模板位置，点击弹出的『立即使用』按钮，即可在个人账户下创建一篇对应的智能文档。


方式三：导入本地文件为智能文档
当前文档首页的『导入』功能，支持将本地的 word 等文件，导入为普通在线文档；如需将本地文件转成智能文档，需要创建一篇智能文档后，
进入到编辑页面，点击右上角菜单栏『≡』，选择『导入文件』，即可将本地的文档、表格、以及在线文档的内容转成智能文档格式。

页头设置

1.1 文档标题
一篇智能文档支持两处标题的设置，两处标题内容可以保持一致，也可以按需设置不同的内容。（大标题对应的是整篇智能文档的内容，而正文中的标题对应的是某个页面节点的内容。）

1.2 文档图标
将悬浮鼠标至文档标题上方，点击浮现的『添加图标』，系统会为文档配置一个随机图标。

如需调整文档图标，可点击下图中『更换图标』，从官方图标中进行选择并更换。当前已经按照文档、QQ、表情、动物、食物等不同类别支持上千种图标样式，全面满足日常办公中的不同需求。


1.3 文档封面
将悬浮鼠标至文档标题上方，点击浮现的『添加封面』，系统会为文档配置一个随机封面。

如需调整文档封面，可点击下图中『更换』，从官方图库中选择喜欢的图片进行封面更换，同时也可以本地导入更贴合文档内容、彰显个性的图片，让文档更加丰富多彩。


页面目录设置

2.1 页面设置
将悬浮鼠标至文档标题上方，点击浮现的『页面设置』，即可进入智能文档的页面设置界面，支持对文档的字体、字号、布局等进行设置。
固定页面宽度：支持自定义调整页面管理和正文之间的宽度，例如，编辑时，更宽的正文页面可以更自由的创作；阅读时，调整页面宽度可以更方便的查看图文、表格等信息，提高信息浏览、编辑的效率。


2.2 页面管理
智能文档提供多页面结构的全新页面管理模式，支持用户对多页面的增删改操作，可新建无限层级和数量页面，并可在页面树自由拖拽，形成页面关系的嵌套，实现文档的统一管理。


2.3 目录管理
除了页面的多层级展示与管理之外，智能文档也支持在右边的侧边栏（如下图）展开当前页面的文档目录。通过点击对应『目录』位置，可以自动跳转到对应的正文内容，方便快速查找和编辑文档内容。


插入丰富内容（基础编辑）

3.1 文档标题编号与排序
在文档标题前直接输入『数字. 空格』，即可为标题添加编号。例如：有序编号1、2、3......，无序编号支持项目列表展示样式；同时，在文档任意一行的行首输入『 / 』，即可触发『快速插入』功能，可直接插入一级、二级、三级.....六级等不同层级的文档标题。


3.2 文档标题（目录）折叠
通过设置文档标题，会自动生成结构清晰的文档目录，针对这些具体的目录节点还可以进行折叠和展开操作。折叠完当前节点的目录后，该节点下的子级节点也会被一起折叠，展开的时候也会一并展开。

3.3 在智能文档中使用引用功能
在任意一行的行首输入『 / 』，或其他任意位置输入『 空格 』+『 / 』 即可触发快速插入 ，在工具栏中选择『 引用文字』后选择即可。在文档中添加引用文字，可以更加详细地描述、解释主要内容，以起到提醒阅读者的作用，让信息传递更加清晰明了。


3.4 在智能文档中使用高亮块功能
高亮块由图标、文本及面板底色填充组成，因其独有的构成形式及亮色填充效果，能够将文档中的重点内容更突出，起到提醒、警示读者的作用，适用于项目背景的介绍、客户案例的总结等应用场景。
在任意一行的行首输入『 / 』，或其他任意位置输入『 空格 』+『 / 』 即可触发快速插入 ，在工具栏中选择『 高亮块』后选择即可。同时，将鼠标放在高亮块的位置，还可通过下方工具栏，对高亮块的字体颜色、边框颜色、背景颜色、图标、字体大小等进行修改。

待办，顾名思义，就是在文档中插入待完成事项，以便于后续追溯完成状态。通过在文档中插入待办，并艾特到指定成员，即可将待办事项指定到具体的责任人，适用于项目管理、会议管理等通用办公场景中。

图片：支持本地图片上传或快捷粘贴复制的图片，并支持文档内预览；

本地文件：支持本地文件的上传，并支持文档内预览；

关于智能文档的常见问题
智能文档和现有doc的区别？
        除了产品形态维度的差异：多页面、内容组件化、编辑体验、重数据应用等；从生产力工具演进的更本质的层次来看，这是我们对新型文档的一次探索，我们把在线文档比喻为「一本词典」，格式丰富、排版工整、纸张段落，是一个行业通用的标准规范（office）；而我们如果想把词典搬到线上，并且做到 100% 还原，这就只能通过 office 来做；智能文档比喻为「一盒乐高」，block 是信息最小单元，所有编辑/应用能力都是一个block，可以根据场景需要去拼装成所需要的形态，不强调编辑能力，更加侧重 block 的使用和形态拓展，更适合想要追求更新的应用文档体验的群体尝试。


智能文档可以调整字体大小、格式、颜色等样式吗？
        智能文档让我们回归到内容本身，而不是文档的排版。整个团队里的文档几乎都是同样的标题就是同样的字体和字号，整齐划一的格式让每个成员都可以简单快速地做出易读的文档。在此设计理念的基础上，智能文档能够对文档的文字样式设置默认、宋体、楷体，对字号设置小号、默认、大号，对文档布局设置紧凑、默认、和宽松。
智能文档的保存按钮在哪里？
      文档没有保存按钮，无需手动保存，网络正常时，文档会实时保存至云端，即时修改即时保存。 网络异常时，你仍可以对文档进行编辑，网络恢复后，离线状态下编辑的内容会自动同步。

一、认识智能表格
       智能表格本质上是一款可以多人在线协同的关系型数据库，只是其交互设计采用类似在线表格的方式。 因此，智能表格同时兼备了业务系统对结构化数据的展示、分析、流转、精细权限控制能力和普通用户对在线表格的低认知门槛。
       而这，这也引出了智能表格与在线表格的最大区别，那就是智能表格会基于列对数据类型进行限制，形成不同的列类型。这一限制听起来简单，但却可以由此衍生出无数在线表格无法具备的功能，或在智能表格上更简便易行的功能。简而言之，列类型让智能表格产生了许多「新花样」。

二、快速上手智能表格
创建与使用
1.1 如何快速创建新的智能表
空白新建：在文档首页，点击『新建』，选择『智能表格』品类，即可跳转智能表格页面，快速新建一篇空白智能表格。


模板新建：在文档首页，点击『新建』，选择『从模板新建』，弹出模板选择界面，点击使用模板后，可以创建一个带有模板内容的智能表格。


1.2 如何迁移现有数据到智能表
生成智能表：如果你正在使用腾讯文档的普通表格，右键普通表底部，会有一个『生成智能表』的选项。


用所选数据新建智能表：在一个普通表格中，选中一片区域后右键，在菜单中可以看到『用所选数据新建智能表』选项，点击『用所选数据新建智能表』，会在同一个表格中中生成一个智能表格，表中的内容是刚才选中的内容。
    

使用「列」类型
2.1 列类型是什么
       智能表格和普通表格的差异之一就是，普通表格可以在任意单元格填写任意内容，也可以对某些单元格进行限制，只能填写某些内容，而智能表格的每一个单元格都只能填写特定的内容，且同一列的所有单元格支持填写的内容类型是统一的，这就是列类型，也即该列支持填写的内容类型。

2.2 如何设置和修改列类型
        点击列头处的下三角，或在列头处右键单击，在弹出的菜单选项中点击「设置列类型」，均可以打开列设置面板，在列设置面板中，可以修改该列的列类型，也可以针对不同的列类型进行格式和配置的设置。


2.3 如何理解不同的列
创建人列：创建人列不可编辑，在一行生成的时候就自动生成，为创建这一行的人员的昵称
最后编辑人列：最后编辑人列不可编辑，在一行生成的时候就自动生成，为修改这一行所有现存列的数据的最后一个人的昵称
创建时间列：创建时间列不可编辑，在一行记录生成的时候就自动生成，为创建这一行记录时的时间
最后编辑时间列：最后编辑时间列不可编辑，在一行记录生成的时候就自动生成，为修改这一行记录所有现存列的数据的最后时间
关联列：在关联列类型中，你可以通过一个智能表的某列数据 ，访问到其他表中与此列数据相关的一行或者多行数据。
引用列：引用列类型在关联列类型的基础上，可以展示关联表的其他列，并可以对列值进行求和、平均值、去重等统计。
公式列：智能表的公式引用数据时，不是引用到具体的单元格，而是引用到列，对于同一条记录而言，就是引用到此列的单元格。

2.4 如何增、删、隐藏、修改列

删除列

 在列头处点击鼠标后移动，可以选中相邻的多列，再点击右键，可以批量删除列
 标题列不可删除
隐藏列
在列头处点击右键，弹出右键菜单，可以看到「隐藏此列」选项，点击「隐藏此列」，选择的列将被隐藏
点击工具栏上的「隐藏列」按钮，弹出侧边栏，在侧边栏可以隐藏列和恢复显示隐藏的列
 在列头处点击鼠标后移动，可以选中相邻的多列，再点击右键，可以批量隐藏列
修改列
双击列头，可以修改列名，列名不可以重复
在列头处右键，弹出列操作菜单，单击「设置列类型」选项，也可以修改列名

画册视图将记录以一个个卡片的形式展示，适合图文并茂的场景，配合表格公告和记录详情效果更佳。


4.4 日历视图
日历视图可以从月的维度总览事项的进展，让用户方便地在智能表中管理自己的日程。


表格公告
       智能表具有表格公告功能，所谓公告，就是在每个智能表的顶部，都可以有一片区域类似普通表格顶部合并单元格的体验，方便用户输入表结构之外的内容。可以在公告内输入用户指引、表格介绍、操作规范等信息。例如：在项目管理场景中，可以介绍项目概况，标注项目重点；在信息统计场景中，可以讲解填写要求，说明截止时间。
       总之，表格公告对表格就像群公告对群聊一样，起到对协作者周知信息的作用。为了满足表格公告在不用场景下的多样化诉求，在表格公告内可以设置字体字色等格式，还可以插入多条链接，设置背景色等。

6.1 如何设置排序

      方式二：在每一列的列头点击右键，再点击「按 A-Z 排序」或「按Z-A 排序」（不同类型的列文案会有差异）可以以该列的内容正向或逆向排序。

6.3 如何设置分组
      方式一：在一个智能表格中，点击工具栏上的「分组选项」，会弹出分组侧边栏，点击添加条件，并选择分组的列，就可以以选择的列的内容进行分组。

      方式二：在每一列的列头点击右键，再点击「按此列分组」，可以以该列的内容分组，如果已有分组条件，该列的条件会放在之前的条件后面。

自动化流程
7.1 自动化流程的概念
自动化可以将工作中重复的事项设置为「流程」，由智能表格来触发运行，帮助用户提升工作效率。任务 由「触发条件」和「执行动作」构成，或者理解为 「……时，就……」。自动化就是类似机器人的角色，帮助你推动、跟进在智能表中记录的工作任务，减少人力投入，提升工作效率。

8.1 内容权限的概念
在智能表格中，可以按工作表、记录、字段来设置「全员权限」，防止内容被误改或泄露。需要为成员分配不同权限时，可以选择「指定成员」来添加额外权限，实现灵活的权限管理。


8.2 内容权限的设置
设置工作表权限：通过侧边栏的“设置工作表权限”按钮，可以设置可编辑、仅浏览等权限。

设置字段权限：右键点击列头，选择“内容权限”，然后设置可编辑、可首次填入、可查看等权限。


8.3 内容权限的使用场景
信息填报时，设置字段权限，允许成员首次填入内容，后续不可修改。
项目管理时，只允许编辑自己是负责人的记录行，避免误改。
门店巡检时，可以为各个角色区分设置权限，让一线员工只能读写自己添加的记录，店长、负责人才能查看更多数据。

       腾讯文档智能表格，是一款新型的数据库电子表格。有别于传统的EXCEl、在线表格等，它具备丰富的列类型和多维的视图展示，可以让你从功能繁复的在线表格中解脱出来，专注于处理数据本身。

❓常见FAQ参考
1.智能表的文档权限、全员权限和额外权限之间的优先级顺序是？
文档权限&全员权限：取小；
全员权限&额外权限：取大，且额外权限一定要大于全员权限，否则页面有提示不生效；
文档权限&额外权限：取小，特别注意排查客户在实际使用过程中，额外权限不生效时，及时排查文档权限的设置。

2.智能表的不同视图的数据源是一样的吗？
是的，智能表格的不同视图数据同源，在同一子表中改动的数据，会在该子表的不同视图中同步展示；同时，仪表盘的数据也与表格数据同源，保持同步修改。

3.智能表可以仅导出「筛选」之后的数据吗？
暂时还不支持，短期内可以先导出到本地后再进行相关筛选；
如果数据量非常大，可以先将筛选后的数据，复制到另一个智能表之后，再进行导出

4.智能表的行、列、单元格限制分别是多少呢？超过后支持单独申请提高限制吗？
智能表当前的不同限制如下：
一个智能表格文件最多256个工作表/sheet；
行数 ：4万；列数 ：150列；单元格数： 80万
图片数每个单元格：9张
文件数每个单元格：9个
超出后暂不支持提升上限，如果数据量过大超出限制，建议先按日期等维度拆表进行管理（同时产品侧正在优化支持更高级别的单元格数量）

5.智能表「电话」列可以输入哪些电话格式呢？
「电话」字段的编辑逻辑，可输入任意位数的数字，“+”(最多输入一个)、“_”以及中英括号(最多输入一对)，中英逗号(最多输入一个)和任意数量的空格。

6.智能表的「关联」字段为什么不能选择展示「人员」姓名？
目前支持的是「文本、数字、链接、日期、电话、邮箱、进度、公式、货币、自动编号」类型的列


7.数字卡可以只引用某一个单元格的数据吗？
方式一：数字卡筛选满足条件的记录后，选择展示对应「字段」的值即可；

暂不支持插入图片、以及插入变量包含图片类型的字段，产品优化评估中


9.智能表格中设置权限的时候，怎样使部门领导仅能看本部门员工填写的信息？
可以在智能表内添加一列作为部门信息，再给领导添加额外权限，编辑/查看范围设置为仅满足条件可查看/编辑，否则不可查看。
  

智能表如何实现不同表格之间的信息自动同步？
可以通过在不同的子表之间创建关联列+查找引用列，实现在子表A中改动数据时，子表B的数据同步改动。

智能表如何实现每个月「最后一天」的自动化通知效果？例如2月份只有28天
如果用户设置了每月 31日触发自动化流程，而某个月只有 28 天或 30 天，系统应该自动调整触发时间，避免触发失败。、
若用户设置了每月31日触发，则对于只有28天的月份以及30天的月份，自动调整为月末触发。
若用户设置了每月30、29日触发，则对于只有28天的月份自动调整为月末触发。

谁可以修改内容权限？
只有文档管理员、超级管理员可以修改内容权限设置，其他成员只能查看与自己相关的权限设置。

文档成员权限与内容权限有什么区别？
文档成员权限高于内容权限，比如文档成员权限是仅浏览，内容权限是可编辑，此时该成员是没有编辑权限的。

内容权限的指定成员额外权限可以设置多少组？
20组。

设置内容权限后，公式、引用字段会按什么权限来计算呢？
为了保护内容安全，公式或引用字段使用到其他字段的内容时，会按照当前查看人的权限来计算。成员如果没有对应字段的权限时，计算结果将和管理员看到的结果有区别。

在单选、多选中添加选项，需要设置什么内容权限？
添加或修改选项，会影响字段的设置，需要工作表的「全部权限」。

我为指定成员设置了额外权限：部分内容不可查看，为什么他们还能看到？
请按以下步骤检查：
1.检查全员权限是否配置了该内容可查看。
成员会同时获得全员权限和指定成员的额外权限，所以全员权限开放了权限后，成员也可查看。
2.检查指定成员记录权限的条件设置。
如果设置了多个条件，需检查是否设置为「任一」条件。设置为任一条件时，成员满足任何一个条件都可获得权限。
3.检查其他指定成员额外权限的人员配置。
请检查成员是否配置了多个额外权限，多个权限会同时生效，重复配置时以较高权限为准。指定成员权限中还可能设置了部门，成员在部门内也会获得权限。

智能表的内容：在哪里可以设置去重？
可以设置分组，然后根据自己不想要内容进行删除或者重新分类（进行分类以后，那么就可以看得出来那些是重复的，例如相同名字/企业/进度/日期等）

多个收集表可以汇总到一个智能表中吗？
可以，先用A收集表关联到新的智能表格，再用B收集表关联到已有的智能表，两个收集表的统计结果会以智能表的两个子表呈现。

智能表「电话」列可以输入哪些电话格式呢？
「电话」字段的编辑逻辑，可输入任意位数的数字，“+”(最多输入一个)、“_”以及中英括号(最多输入一对)，中英逗号(最多输入一个)和任意数量的空格。

智能表开启了选项字段「允许填写时新增选项」，为什么在填写的时候还是无法新增选项呢？
场景一：设置了内容权限的情况下，当前会影响「新增选项」功能的使用；
场景二：开启全部权限，如果用户通过表单「链接」来进行填写，也无法「新增选项」（表格内可以）——表单视图独立链接只能执行一行的请求，不支持对「列」进行配置修改。
—— 使用建议：新增备注列，并在原选项字段种新增「其他」，再定期由管理员来进行配置修改


设置了只有本人才能编辑的内容权限后，就无法批量新增行了吗？
当字段类型为人员，内容权限设置了人员等于自己可编辑，其他人不可查看的时候，文档成员无法批量新增行。
此时将字段类型改为创建人就要批量新增了。（内容权限也对应调整为等于创建人）

分组可以一键折叠和展开吗？
支持，选中分组右键。


智能表内容显示不全时怎么办？
      方式一：拉长列宽。

      方式二：调整行高。

三、文档权限设置说明说明：目前腾讯文档私有化版本，暂不支持企业外部的协作，因此文档的权限设置主要基于组织范围内。同时，除了基础的如在线文档权限设置之外，在线表格还涉及到精细到单元格的权限保护，也会在此部分进行相关补充说明。1、加入规则加入规则：指企业内成员通过文档链接，访问文档时的默认权限，目前支持四种不同的设置，分别为：宽松：可直接加入，加入时可获得编辑权限较宽松：可直接加入，加入时可获得浏览权限严格：需申请加入，由文档成员审批权限较严格：需申请加入，仅由管理员审批权限注：文档的默认权限，遵循管理后台统一设置的「文档默认权限」规则，文档创建者（超级管理员）可在此基础上单独修改文档权限。2、文档成员文档成员支持按照部门、成员的维度来进行添加或调整，并支持在这个页面进行搜索、筛选以及批量修改文档成员的权限，同时，在个别权限冲突的场景下，权限优先遵循的逻辑为：场景一：当文档同时添加A员工、及A员工所在B部门为文档成员，且权限不一致时：表现：保留A员工的权限；逻辑：个人权限>部门权限。场景二：当文档同时添加A员工，但A员工同时处于B、C两个（或以上）部门，且各部门权限不一致时：独立的两个（或多个）部门，默认取最大值（1.6版本更新）；存在上下级关系两个（或多个）部门，默认取最小节点部门的权限值（1.6版本更新）。3、安全设置安全设置，包括了文档水印设置及更严格的权限管控，分别为：文档水印：如果管理后台设置了「强制水印」，则文档管理员则无法再次设置文档的水印功能；如果文档本身已经设置了水印，再开启后台「强制水印」，水印不会叠加，仍旧是「强制水印」优先生效。同时前端设置水印入口置灰，管理员无法操作；禁止可预览成员发表评论：开启后，只有预览权限的成员则无法发表评论；禁止成员复制内容：可以「不限制」、「可预览」、「所有人」不能复制、打印、导出或创建副本，严格管控文档内容的权限，有效防止信息泄露的风险。4、其他：在线表格的区域保护范围设置腾讯文档支持保护某个范围的单元格或保护某个工作表，保护区域的内容其他协作者无法修改或者无法查看，目前仅支持电脑端进行设置，设置成功后，不可编辑或者不可查看在电脑和手机端都能生效。区域保护范围支持添加部门（1.8更新）或成员，均可生效；当同一区域范围的权限设置冲突时，如A规则成员可编辑，B规则成员可查看时，优先最小的权限；文档管理员仅可以修改自己有编辑权限的保护范围，当有「转让文档所有者」需求时注意权限变化；普通成员（预览、编辑权限）无法新建保护范围的规则，相关功能入口会置灰，且无法查看历史保护规则。腾讯文档企业版私有化-文档权限说明一、权限来源二、权限逻辑1、权限概览个人空间：即「我的文档」，可将个人名下的文档进行统一管理，权限始终与文档权限一致共享空间：支持从空间-文件夹-文档的分级管理，权限默认继承并支持灵活调整，满足团队文档管理需求注：更多权限表现产品补充可查看产品补充2、不同「人」的权限逻辑当员工A仅属于部门X：员工权限始终跟随部门权限变化而变化；当员工A同时属于两个独立的部门X和Y：员工权限默认取两个权限中的最大值（1.6版本更新）；当员工A同时属于有上下级关系的部门X和Y：员工权限默认取最小节点部门的权限值（1.6版本更新）；注：当成员权限与部门权限同时存在，且权限设置冲突时，始终默认个人权限>部门权限。3、不同「资源」的权限逻辑3.1 不同资源之间的关系图共享空间：分成普通共享空间和部门共享空间，拥有空间权限的成员为「空间成员」；文件夹：共享空间下创建文件夹，文件夹权限包括「共享成员」+「空间成员」；文档：在空间及文件夹下均可创建文档，文档权限包括「共享成员」+「空间成员」+「文档独立成员」。注：当前空间/文件夹下的文档权限列表中，不会区分具体成员的类型，当权限冲突时：
优先按照个人权限>部门权限的逻辑执行，权限面板所见即所得；
当权限节点同时为个人或部门时，按照独立文档成员>共享成员>空间成员>加入规则（1.6版本更新）。3.2 不同资源之间的鉴权逻辑共享空间→文件夹：默认继承空间权限；修改空间成员权限后，不再往下继承；文件夹→子文件夹：默认继承空间权限+文件夹权限；修改空间成员权限后，仅继承文件夹的共享成员权限；共享空间→文档：默认继承空间权限；修改空间成员后，直接在空间下创建的文档仍可继承修改后的权限；文件夹→文档：默认继承空间权限+文件夹权限；修改空间成员权限后，仅继承文件夹的共享成员权限。

三、个人空间1、个人空间的入口个人空间：即「我的文档」，点击「新建」即可快速创建新的文件夹/子文件夹，在文件夹下创建的文档，会自动归类到指定的文件夹目录。2、移动文档到个人空间个人创建的文档会默认归属在「我的文档」，同时支持将个人文档迁移到个人空间下的指定文件夹。3、个人空间的权限说明第一层：空间权限，仅本人，且不支持添加成员；第二层：文件夹权限，仅本人，且不支持添加成员；第三层：文档权限，默认创建者和超管都是本人，支持对文档进行权限管理，并添加、删除、修改成员权限。四、共享空间1、共享空间的入口共享空间：进入腾讯文档的首页，在左侧可见「共享空间」的入口，所有「我创建」的、和「我加入」的共享空间，都会在这个列表进行展示，并支持逐层展开空间下的各级目录。点击「新建」，即可创建一个新的共享空间，并在空间下新增文件夹、及文档等丰富内容。2、设置空间权限选择对应「共享空间」，进入「权限管理」设置页面，可以为共享空间设置第一层权限，即空间权限，在这一层添加的成员，为「空间成员」。空间成员：空间超管：可以添加任意成员权限为管理员、可编辑、可下载、仅预览等多种权限。空间管理员：可以添加任意成员权限为管理员之外的权限，即可配置小于自己的权限；普通成员：在「安全设置」未开启「仅管理员可增减成员和修改查看范围」功能时，默认可以添加小于等于自己的权限。安全设置：保密模式：开启保密模式后，会强制开启：新建文档/文件夹的默认加入规则为「仅由管理员审批权限」、仅管理员可增减成员和修改查看范围、以及开启文档预览水印；新建文档/文件夹的默认加入规则：支持配置四种不同严格程度的加入规则，同时，文件夹/文档管理员可在此基础上单独修改某节点的默认加入规则；仅管理员可增减成员和修改查看范围：开启，则控制只有「超管」+「管理员」角色可以添加空间成员；关闭，则空间的所有成员都可以添加成员，并赋予小于等于自己的空间权限（管理员不能添加其他人为管理员角色）；开启文档预览水印：开启后，水印显示「文档管理者水印」+姓名，并提示“空间已开启文档水印，文档所有者不可关闭“；不开启，则由文档管理员单独设置文档水印；启用成员邀请链接/二维码：启用后，成员可以在「权限管理」处复制「邀请链接」和二维码；并可在此基础上启用「通过链接/二维码加入空间无需审批」，都开启后，空间外的成员可直接通过链接加入空间获得可预览权限。3、设置文件夹权限进入「共享空间」，可在页面通过「新建」创建空间下的文件夹/子文件夹。选择对应「文件夹」，进入「权限管理」设置页面，可为文件夹设置第二层权限，即文件夹权限，在这一层添加的成员，为「共享成员」。文件夹成员：共享成员：通过「添加成员」，可为当前文件夹节点添加额外权限，且共享成员权限可往下持续继承；空间成员：默认继承上级，即共享空间的成员权限；当共享空间内文件夹添加了「共享成员」，即可在共享空间内文件夹权限管理面板手动断开继承空间成员的权限。如需恢复文件夹与空间的继承关系，可点击“恢复”。恢复后，此文件夹成员将会被清空，并重新同步空间成员权限;加入规则：默认共享空间设置的「新建文档/文件夹的默认加入规则」，管理员也可手动修改。4、设置文档权限进入「共享空间」下的「文件夹」，可在页面通过「新建」创建新的在线文档、或上传本地文件。选择对应文档/文件，进入「权限管理」设置页面，可为文档设置第三层权限，即文档权限，在这一层添加的成员，为「文档独立成员」。文档成员：文档独立成员：通过「添加成员」，可为当前文档添加额外权限，权限不影响其他文档、文件夹；共享成员：在面板中，不会展示共享成员的角色，但默认会继承上级节点的共享成员；空间成员：在面板中，不会展示空间成员的角色，但默认跟随上级节点的权限（即：上级继承空间成员权限，则文档也继承；反之，则不继承）。加入规则：默认共享空间设置的「新建文档/文件夹的默认加入规则」，管理员也可手动修改。安全设置：与文档的安全设置一致，包括水印、评论、复制功能限制等。四、权限常见FAQ1、部门共享空间和普通共享空间有什么区别？样式上：部门共享空间会在前端展示多一个「部门」标签，普通部门空间则没有；超管权限：部门空间默认部门负责人为空间超管，普通空间则默认创建者为空间超管；成员权限：部门空间成员权限默认跟随部门成员变化而变化，普通空间则根据邀请和添加的成员变化而变化。2、将个人文档迁移到共享空间的某个文件夹下，权限如何变化？超管权限：文档所有者变更为新的共享空间超管，原所有者角色降级为文档管理员，但创建者信息不变；其他权限：取文档本身权限和文件夹权限的并集，例如文档权限为A+B，文件夹权限为C+D，则迁移之后的文档权限为A+B+C+D。3、将原本在A共享空间下的文档迁移到B共享空间，权限如何变化？迁出：文档从A共享空间迁出，原本继承空间+文件夹的权限会默认移除，仅保留单独为文档添加的权限；迁入：同问题2，取文档本身权限和文件夹权限的并集。4、添加文档的快捷方式到共享空间的某个文件夹下，权限如何变化？快捷方式：与迁移原文档不同，如果只迁移文档的快捷方式，则与共享空间和文件夹的权限都没有关联；迁移副本：生成副本则生成了一个新的文档链接，仅由副本继承空间和文件夹的权限。5、如果同一成员在文档、文件夹、和共享空间的权限都不一致，优先哪个权限？权限优先级：文档独立成员>共享成员>空间成员>加入规则，因此第一优先文档的权限。6、如果成员和TA所处部门的权限不一致，优先哪个权限？权限优先级：个人权限>部门权限，因此第一优先个人的权限。7、如果只设置了部门权限，在该部门的员工权限如何变化？当员工仅属于一个部门：员工权限始终跟随部门，权限变化而变化；当员工同时属于两个独立的部门：员工权限默认，取两者权限的最大值。（当部门存在上下级关系时，取最小部门节点的权限值）
